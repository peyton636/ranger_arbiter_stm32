# 传感器测量与运动判断 — 完整逻辑说明

从 **main()** 出发，描述四路测距模组的 **采集 → 数据转换 → 避障运动决策 → CAN 下发** 全链路。

> 主要源码：`User/main.c`、`APP/distance_sensor/`、`APP/time/time.c`、`APP/arbiter/arbiter.c`

---

## 1. 总览（一张图）

```text
main() 初始化 DistanceSensor + Arbiter
        |
        v
  [并行] TIM4 20ms -> DistanceSensor_Process -> PB10 触发测距
        |
        v
  [并行] USART3 中断 -> 10 字节帧 -> ds_data (原始)
        |
        v
  while(1) SensorData_ShowScreen() 约 10ms/轮
        |
        +- NewData -> 滤波 -> g_front/back/left/right_dist_mm
        |
        +- Motion_ControlByDistance()
        |      Arbiter_SetObstacleDistances(四向距离)
        |      Arbiter_Process()  -> 模式 + 速度/转角输出
        |      Arbiter_SendToSTM32A() -> CAN 0x111
        |
        +- Print / LCD 显示（调试与人机界面）
```

## 2. 从 main 开始（文字版全流程）

### 2.1 启动阶段（只执行一次）

| 步骤 | 代码 | 作用 |
|------|------|------|
| 1 | `Hardware_Check()` | 自检；内部 `TIM4_Init(200,8399)` 启动 **20ms** 定时器 |
| 2 | `DistanceSensor_Init()` | USART3(PB10/PB11) 9600，开接收中断，滤波器复位 |
| 3 | `Arbiter_Init()` | 仲裁状态机、四向距离初值 UNKNOWN、输出 v=0 |
| 4 | `CAN1_Init_RangerMini()` | 底盘 CAN |
| 5 | `while(1) SensorData_ShowScreen()` | 主循环，不再返回 |

（同文件还初始化 Jetson 串口、GPS 等；**运动意图**来自 Jetson，**障碍距离**来自测距传感器。）

### 2.2 背景任务（与主循环并行）

**A. TIM4 每 20ms — `DistanceSensor_Process()`**

- 每 **240ms**（12×20ms，`DS_TRIG_EVERY_N_TICKS=12`）：
  1. 关 USART3 接收中断；
  2. PB10 改 GPIO 并 **拉低**（触发 E08 模组测距）；
  3. 下一拍恢复 PB10 为 UART TX，再开中断。
- 收包过程中若 **200ms** 无新字节 → `ds_log_rx_timeout`，主循环打印 `[DS] RX timeout`。

**B. USART3 中断 — `USART3_IRQHandler()`**

- 同步收 **10 字节** UART 帧（见第 3 节）；
- 校验通过 → 写入 `ds_data`，`ds_frame_ready = 1`。

### 2.3 主循环每一轮 — `SensorData_ShowScreen()`（传感器+运动相关）

| 顺序 | 动作 | 说明 |
|------|------|------|
| 1 | `DistanceSensor_DrainLog()` | 打印收包超时 |
| 2 | `DistanceSensor_NewData()` | 若有新帧：清标志 + **四路滤波** |
| 3 | 若 `sensor_updated && ds->valid` | 更新全局四向距离（见 4.3 节） |
| 4 | `Motion_ControlByDistance()` | **注入距离 → 仲裁 → 发 CAN**（见第 5 节） |
| 5 | 有新帧时 | `DistanceSensor_Print()` + LCD 刷新 Count/四路距离 |
| 6 | `delay_ms(10)` | 本轮结束 |

## 4. 数据转换（原始 → 可用距离）

整条转换链：

```text
UART 原始 u16
    -> [中断] error 判定 + FAILSAFE 归一化  -> ds_data.dist[], error[]
    -> [NewData] NormalizedMm(i)            -> 滤波器输入
    -> [滤波] 窗口/中位数/野值门控           -> stable_mm (每路)
    -> GetArbiterMm(i) == GetFilteredMm(i)  -> g_*_dist_mm
    -> Arbiter_SetObstacleDistances()       -> 运动判断
```

### 4.1 第一层：中断内原始 → error + dist

| 模组原始值 | error | 存入 dist |
|------------|-------|-----------|
| 30 ~ 59999 mm | NONE | 原值 |
| 0xEEEE | CHKFAIL | 0 |
| 0xFFFF / 0 / >59999 | TIMEOUT | 0 |

### 4.2 第二层：滤波（`DistanceSensor_FilterOnNewFrame`）

每收到一帧，四路各自维护滑动窗口：

| 参数 | 值 | 含义 |
|------|-----|------|
| 窗口 | 2000 ms，最多 5 样本 | 时间滑动 |
| 有效范围 | 30 ~ 3000 mm | 窗外样本丢弃 |
| 野值 | 相对中位数突跳 >800 mm | 丢弃 |
| 输出 | 有效样本 **中位数** | `stable_mm` |
| 运动跟踪 | 相对 stable 变化 >150 mm | 清窗快速跟变 |

### 4.3 第三层：main 写入全局变量

```c
// sensor_updated && ds->valid 时：
g_front_dist_mm = DistanceSensor_GetArbiterMm(DS_IDX_FRONT);  // IF1 滤波值
g_back_dist_mm  = DistanceSensor_GetArbiterMm(DS_IDX_BACK);
g_left_dist_mm  = DistanceSensor_GetArbiterMm(DS_IDX_LEFT);
g_right_dist_mm = DistanceSensor_GetArbiterMm(DS_IDX_RIGHT);
g_nearest_dist_mm = DistanceSensor_GetFilteredMinDistMm();   // 蜂鸣用
```

**注意**：送入运动仲裁的是 **滤波后** 的四向距离，不是单帧 raw。

### 4.4 距离语义（传感器与仲裁共用）

| 数值 | 含义 |
|------|------|
| 0xFFFF (65535) | UNKNOWN — 尚未有有效滤波/未更新 |
| 0 | FAILSAFE — 超时或校验失败归一化结果 |
| 1 ~ 59999 | 正常毫米值 |

## 5. 运动判断（根据传感器距离）

入口：`Motion_ControlByDistance()`（main.c，每轮主循环调用）

```text
Arbiter_SetObstacleDistances(front, back, left, right)
        |
        v
Arbiter_Process()          // 按当前模式决策 output.v / steering / omega
        |
        v
Arbiter_SendToSTM32A()     // 约每 20ms 打包 CAN ID 0x111 发底盘
```

### 5.1 距离注入 — `Arbiter_SetObstacleDistances()`

1. 四向距离 `Sanitize`（超 59999 → UNKNOWN）；
2. 置 **valid 掩码**（哪几路数据有效）；
3. 算 **nearest_dist** = 四向有效最小值；
4. 算 **obstacle_detected**：最近距 <= **动态 FAR** → 认为有障碍，需限速/避障。

### 5.2 距离阈值

**固定阈值（蜂鸣等）**

| 宏 | mm | 用途 |
|----|-----|------|
| OBSTACLE_NEAR | 60 | 危险区（固定硬判定参考） |
| OBSTACLE_WARN | 120 | 蜂鸣预警 |
| OBSTACLE_FAR | 180 | 固定限速参考 |

**动态阈值（策略层，随速度变）**

- **FAR_dyn** = 反应距离 + 制动距离 + 余量（约 250~800 mm）
- **NEAR_dyn** ≈ FAR_dyn / 3（约 60~200 mm）
- 速度越高，FAR/NEAR 越大，越早进入障碍/危险判定

策略里：

- `obs_*`：该方向距离 <= FAR_dyn → **有障碍**
- `danger_*`：该方向距离 <= NEAR_dyn → **危险**
- 某方向 **invalid/UNKNOWN** → 策略侧当 **有障/危险**（保守）

### 5.3 仲裁模式状态机

```text
                    心跳丢失
NORMAL  ------------------------> DEGRADED
  |  (obstacle_detected            (本地降级策略)
  |   或前向无效)                      |
  v                                    |
SPEED_LIMIT <-------------------------+
  |  四向全危险                         |
  v                                    |
EMERGENCY <----------------------------+
  |  稳定 1s + 心跳恢复                 |
  v                                    |
RECOVERING --------------------------> NORMAL
```

| 模式 | 传感器如何影响运动 |
|------|------------------|
| **NORMAL** | 透传 Jetson 速度；若 `obstacle_detected` 或前向无效 → 切 **SPEED_LIMIT** |
| **SPEED_LIMIT** | 按 Jetson **运动意图** + 四向距离做限速/绕行；前方空且有效 → 回 NORMAL |
| **DEGRADED** | Jetson 心跳丢失；纯本地按传感器 **选最空方向** 避障 |
| **EMERGENCY** | 四向均 NEAR 危险 → v=0；侧向仍有空间则退出到 DEGRADED/RECOVERING |
| **RECOVERING** | 停车等待 1s 稳定后回 NORMAL |

### 5.4 策略总入口 — `Arbiter_ProcessJetsonIntentPolicy()`

仅在 **SPEED_LIMIT** 或 **DEGRADED** 模式下，按 Jetson **运动意图** + 四向传感器距离决策。

流程：

1. 用当前 **参考速度** `v_ref = max(|Jetson.v|, |底盘反馈线速度|)` 计算 **NEAR_dyn / FAR_dyn**；
2. 四向分别算 `obs_*`（<=FAR 有障）、`danger_*`（<=NEAR 危险）；
3. 解析意图 → 调用对应 `Apply*Intent()`；
4. 输出 `arb_state.output.v / steering / omega`；
5. SPEED_LIMIT 末尾再调 `TryRecoverJetsonSpeedInSpeedLimit()` 尝试同向提速。

---

### 5.5 距离 → 速度比例控制（线性缩放）

**用途**：同向直行（前进/后退）时，根据 **运动方向上的传感器距离** 把 Jetson 目标速度按比例缩小，而不是只有 0/全速两档。

**核心函数链**：

```text
Arbiter_JetsonForwardSpeed(dist_front)   // 或 BackwardSpeed(dist_back)
    -> Arbiter_ScaleJetsonVByDist(v_cmd, dist_mm, valid)
        -> Arbiter_CalcDistFactorPct()  得到 0~100%
        -> output.v = v_cmd * pct / 100
        -> Arbiter_ClampSpeed()         限幅 ±1000 mm/s
```

**距离因子 pct（`Arbiter_CalcDistFactorPct`）** — 分段线性：

| 距离 dist | pct | 输出速度 |
|-----------|-----|----------|
| invalid / UNKNOWN | 0 | **0**（视为最危险） |
| dist <= NEAR_dyn | 0 | **0** |
| NEAR_dyn < dist < FAR_dyn | 线性插值 | ` (dist-NEAR)/(FAR-NEAR) * 100 ` |
| dist >= FAR_dyn | 100 | **全速**（Jetson 目标速度） |

**公式**：

```text
pct = (dist - NEAR_dyn) * 100 / (FAR_dyn - NEAR_dyn)    // 仅当 NEAR < dist < FAR

v_out = v_jetson * pct / 100
```

**NEAR_dyn / FAR_dyn 随速度变化**（越快，越早减速）：

```text
FAR_dyn = clamp( v_ref * T_delay + v_ref^2 / (2*a_brake) + margin , 250~800 mm )
NEAR_dyn = clamp( FAR_dyn / 3 , 60~200 mm )

其中：T_delay=350ms, a_brake=2000 mm/s^2, margin=100mm
      v_ref = max(|Jetson.v|, |底盘反馈速度|)
```

**数值例子**（假设 v_ref=500mm/s → FAR_dyn≈500, NEAR_dyn≈167）：

| 前方距离 | pct 约 | Jetson v=500 时输出 |
|----------|--------|---------------------|
| 100 mm | 0 | 0 |
| 200 mm | ~10% | ~50 mm/s |
| 333 mm | ~50% | ~250 mm/s |
| 500 mm+ | 100% | 500 mm/s |

**默认兜底速度**（Jetson v 过小或未给时）：

| 场景 | 默认 | 宏 |
|------|------|-----|
| 前进 | 100 mm/s | ARB_CMD_FORWARD_MM_S |
| 后退 | -75 mm/s | ARB_CMD_BACKWARD_MM_S |
| 斜移 | 500 mm/s | ARB_SLIDE_SPEED_MM_S |

**同向提速回归**（`TryRecoverJetsonSpeedInSpeedLimit`）：

- 策略已算出绕行动作后，若当前 output.v 与 Jetson **同向**且 **比** 距离缩放后的目标更慢，则提到 `ScaleJetsonVByDist` 允许的速度；
- **不会**强行把绕行动作改成直行（只提速、不改方向）。

**DEGRADED 断连模式**：

- 选最空方向后，前进/后退仍走 `ScaleJetsonVByDist(默认速度, 该方向距离)`，同样受 NEAR/FAR 线性限速；
- 左/右斜移用固定 `ARB_SLIDE_SPEED_MM_S`，不按距离再缩放。

### 5.6 四向权重 — 选绕行方向（不直接乘速度）

**用途**：当前方（或运动方向）**过近必须绕行**时，在后/左/右（或前/左/右）中选 **“等效距离最大”** 的方向。权重只影响 **选哪边绕**，不直接乘到 v 上。

**加权等效距离**：

```text
weighted_dist = PolicyDist(raw_dist, valid) * weight / 1000

PolicyDist:  UNKNOWN/invalid -> 0（最差）
             FAILSAFE(0)      -> 0
             正常 mm          -> 原值
```

**权重常量**（千分比，1000=1.0）：

| 常量 | 值 | 含义 |
|------|-----|------|
| WEIGHT_FULL | 1000 | 完全信任 |
| WEIGHT_HIGH | 900 | 高 |
| WEIGHT_MEDIUM | 600 | 中 |
| WEIGHT_LOW | 300 | 低 |
| WEIGHT_IGNORE | 200 | 几乎忽略（仍参与比较） |

**按运动意图分配四向权重**（`Arbiter_GetSensorWeights`）：

| 意图 | front | back | left | right | 说明 |
|------|-------|------|------|-------|------|
| 前进 FORWARD | 1000 | 200 | 600 | 600 | 绕行时偏好后/侧方 |
| 后退 BACKWARD | 200 | 1000 | 600 | 600 | 绕行时偏前/侧方 |
| 左移 SLIDE_LEFT | 900 | 300 | 1000 | 200 | 左受阻看右，前后比远近 |
| 右移 SLIDE_RIGHT | 900 | 300 | 200 | 1000 | 右受阻看左 |
| 阿克曼左转 steer>0 | 900 | 300 | 1000 | 300 | 左转关心左侧 |
| 阿克曼右转 steer<0 | 900 | 300 | 300 | 1000 | 右转关心右侧 |
| 阿克曼直行 steer=0 | 同前进 | | | | |

**选方向示例 — 前进受阻（rule 6）**：

```text
比较 weighted: back vs left vs right
取最大者 -> 0:后退  1:左斜移90°  2:右斜移90°
后退速度 = JetsonBackwardSpeed(dist_back)  // 仍按距离缩放
斜移速度 = JetsonSlideSpeed()            // 一般固定 500，不随距离缩
```

### 5.7 各意图：距离分区 + 权重 + 速度 — 对照表

**前进意图 `ApplyForwardIntent`**（看 **前方** dist=f）：

| 条件 | rule | 动作 | 速度来源 |
|------|------|------|----------|
| 前方无障 !obs_f && !danger_f | 12 | 阿克曼直行 | JetsonForwardSpeed(f) 全比例缩放 |
| 前方有障但未危险 near<f<=far | 11 | 阿克曼直行 | JetsonForwardSpeed(f) 线性减速 |
| 前方危险或 f<=near | 6 | 加权选后/左/右绕行 | 后退用 BackwardSpeed(b)；斜移用 SlideSpeed |

**后退意图 `ApplyBackwardIntent`**（看 **后方** dist=b）：对称，rule 7 + 加权选前/左/右。

**左移意图 `ApplySlideLeftIntent`**（看 **左侧**）：

| 条件 | 动作 |
|------|------|
| 左侧无障 | 左斜移 SlideSpeed |
| 左侧有障、右侧无障 | 改右斜移 |
| 左右都有障 | 比较加权 front vs back，选前进或后退（带 ScaleJetsonVByDist） |

**右移意图**：与左移对称。

**自旋 `ApplySpinIntent`**：四向全危险则 omega=0，否则透传 Jetson omega（不按距离缩放线速度）。

### 5.8 距离、权重、速度 — 三者关系小结

```text
                    +------------------+
  传感器四向 mm ---->| SetObstacleDistances |
                    +------------------+
                              |
              +---------------+---------------+
              |                               |
     判断 obs/danger                  nearest_dist
     (NEAR/FAR 分区)                  (切 NORMAL/SPEED_LIMIT)
              |                               |
              v                               v
     选动作：直行 or 绕行              模式状态机
              |                               |
     直行 ----+---- 绕行                     |
              |         |                    |
              v         v                    |
     ScaleJetsonVByDist   PickFreest*Weighted |
     (距离线性 -> v)      (权重 -> 选方向)      |
              |         |                    |
              +----+----+                    |
                   v                         v
              arb_state.output.v  -----> CAN 0x111
```

- **距离 → 速度**：`ScaleJetsonVByDist`，NEAR~FAR 之间 **线性 0~100%**；
- **权重 → 方向**：只在 **必须绕行** 时使用，选加权距离最大的侧向；
- **动态 NEAR/FAR**：速度越快，减速区越早开始。

串口可观察 `[SAFE] mode=... v_ref=... danger=NEAR limit=FAR F=... obs=... dng=...`（约 500ms 一条）。

---

### 5.9 CAN 输出 — `Arbiter_SendToSTM32A()`

- 周期：主循环约 10ms × `ARBITER_CAN_DIV(2)` ≈ **20ms** 一发；
- ID **0x111**：线速度 mm/s、转角 0.001rad、自旋速度等；
- 若 `emergency`、遥控模式、底盘模式切换中 → 强制发 **零速**。

### 5.6 强制停车（与传感器并行）

- **KEY0** 可置 `g_force_stop_enable`：`Motion_ControlByDistance()` 忽略策略，直接 v=0 发 CAN。

## 6. 端到端示例（前进遇障）

1. TIM4 触发 → 模组回帧 IF1=400mm, IF2=2000mm, IF3=1500mm, IF4=1500mm；
2. 中断解析 → 滤波 → `g_front_dist_mm=400` 等；
3. `SetObstacleDistances` → 最近 400mm < FAR_dyn → `obstacle_detected=1`；
4. NORMAL → 切 **SPEED_LIMIT**；
5. Jetson 意图=前进，前方 obs_f=1 → 若 400mm 在 NEAR~FAR 之间则 **限速前进**；若 <NEAR 则 **选左/右/后最空方向斜移或后退**；
6. CAN 0x111 下发仲裁后的 v、steering；
7. LCD 显示 `400 mm (398)`，串口 `[DS] IF1: 400 mm`。

## 9. 常见问题

| 现象 | 可能原因 |
|------|----------|
| `[DS TEST]`：`trig` 涨、`rx=0`、`skip=0`、`raw=(none)` | **E08 5V 供电或 GND 共地**（见 [`硬件连接与通信协议.md` §7.1](./硬件连接与通信协议.md#71-供电必读)、[`以太网与WiFi接入方案.md` §15.11](./以太网与WiFi接入方案.md#1511-测距无数据e08-供电问题2026-06-实验室确认)） |
| 同一固件有时 `rx>0` 有时 `rx=0` | 杜邦线松动、5V 压降、电源负载过大 |
| `rx>0` 但 `FF=0`、`skip≈rx` | 波特率非 9600、TX/RX 接反、供电不稳致乱码 |
| `FF≥1`、`chk>0`、`ok=0` | 帧头对但校验失败：干扰、线太长、接触电阻 |
| 开以太网后测距 `--` 但 `rx>0` | **T1** `App_EthPoll` 长时间关中断丢字节（见以太网文档 §15.4） |
| 有 [DS] 但车不动 | Jetson 未发有效指令(v=0) 或 NORMAL 未切 SPEED_LIMIT |
| 一直 DEGRADED | Jetson 心跳超时 |
| 四向 TIMEOUT 急停 | 四向均 FAILSAFE/NEAR → EMERGENCY |
| IF 方向反了 | 改 main.c 中 DS_IDX_FRONT/BACK/LEFT/RIGHT 映射 |

### 9.1 独占测距联调（`APP_SENSOR_TEST_ONLY`）

`APP/freertos/app_boot.h` 中 `APP_SENSOR_TEST_ONLY=1` 时关闭 ETH/GPS/CAN/Jetson，仅保留 TIM4 触发 + USART3 + `[DS TEST]` 日志，用于区分 **软件/以太网** 与 **硬件供电** 问题。测距正常后改回 `0` 再开全功能。
