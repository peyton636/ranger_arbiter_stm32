本文档描述 **麒麟 F407 综合实验程序** 的 **硬件接线、引脚、速率与系统拓扑**。  
**Jetson 上位机** 与 F407 之间 **CAN 与 RS232 二选一**（编译期切换）；具体帧格式见独立协议文档：

| 链路 | 协议文档 |
|------|----------|
| **CAN2**（默认） | [Jetson_CAN协议.md](./Jetson_CAN协议.md) |
| **USART2 / RS232** | [Jetson_RS232协议.md](./Jetson_RS232协议.md) |

本文档 **§5 起** 为 **底盘 CAN1**（F407 ↔ RANGER MINI）协议详表。

## 🔀 Jetson 链路切换（CAN / RS232 二选一）

**同一固件只启用一条链路**，改宏 → 接线 → **重新编译烧录**（运行时不能双发）。

| | **CAN 模式（默认）** | **RS232 模式** |
|:--|:--|:--|
| **宏** | `JETSON_LINK_CAN` **= 1** | **= 0** |
| **文件** | `APP/jetson_can/jetson_can.h` | 同上 |
| **接线** | **PB5/PB6** + TJA1050 → Jetson USB-CAN | **PA2↔Jetson RX，PA3↔Jetson TX**，GND 共地 |
| **速率** | 500 kbps | 115200 8N1 |

```c
#define JETSON_LINK_CAN  1   /* 1=CAN2，0=USART2 */
```

**四步切换：** ① 改上面宏 → ② 按模式接线 → ③ 全量编译烧录 → ④ Jetson 侧用对应解析器（CAN 或串口）

详表见下文 **§2.2**；CAN 协议 → [Jetson_CAN协议.md](./Jetson_CAN协议.md) · RS232 协议 → [Jetson_RS232协议.md](./Jetson_RS232协议.md)

</div>

## 1. 系统架构

### 1.1 拓扑图（修正版）

原 ASCII 图有三处易误解：**① `CAN 500K` 误标在测距与 F407 之间**（CAN 只连底盘）；**② Jetson 只画了单向**（实际双向收发）；**③ PC/GPS 悬空**（实际都接在同一块 F407 上）。

```text
                    +----------------------+
                    |   Jetson 上位机       |
                    | (运动规划 / 视觉)     |
                    +-----------+----------+
                                |
         ┌──────────────────────┴──────────────────────┐
         │  二选一（JETSON_LINK_CAN，见 §2.2）           │
         │  A) CAN2: PB5/PB6 + TJA1050 → USB-CAN 500k   │
         │  B) RS232: PA2/PA3 ↔ Jetson TTL 115200 8N1   │
         └──────────────────────┬──────────────────────┘
                                |
    +-------------+             |             +---------------------------+
    | 四路测距 E08 | PB11 RX ◄──┼── UART 收帧 |                           |
    |  转接模组    | PB10 TX/触发┼────────────►|  麒麟 F407 开发板          |
    +-------------+   USART3    |             |  (本工程 = 逻辑上 STM32 B)   |
         9600 8N1               |             |  仲裁 · 避障 · LCD · 蜂鸣   |
                                |             +----+------+--------+--------+
    +-------------+             |                  |      |        |
    |  PC 调试     | USB/UART ───┼──────────────────┘      |        |
    |  (printf)    |  USART1 PA9/10 115200                 |        |
    +-------------+             |                           |        |
                                |                  CAN1 PA11/12     |
    +-------------+             |                  500 kbps  │        |
    |  GPS 模块    | PC6/7 ──────┼──────────────────────────┘        |
    +-------------+  USART6     |                           |        |
         9600→115200             |                           v        |
                                |             +---------------------------+
                                |             |  STM32 A / 底盘控制器      |
                                |             |  RANGER MINI 3.0          |
                                |             |  电机 · BMS · 里程计       |
                                +────────────►|  (CAN 总线，非串口)        |
                                              +---------------------------+
```

### 1.2 物理板 vs 逻辑角色

| 实体 | 是什么 | 在本图中的位置 |
|------|--------|----------------|
| **麒麟 F407 开发板** | 你手边这块实验板，程序跑在这里 | 图中中央大块 |
| **STM32 B（逻辑名）** | 代码里的「仲裁板」角色，**就是这块 F407**，不是第二块 MCU | 与 F407 同一块板 |
| **STM32 A / 底盘** | 小车上 RANGER MINI 自带的底盘控制器 | 通过 **CAN** 与 F407 相连 |
| **Jetson** | 独立上位机（如 Orin/NX） | **CAN2 或 USART2 二选一** 与 F407 相连，**不直连底盘** |
| **四路测距** | E08 转接 + 4 探头 | 通过 **USART3** 接 F407；**不走 CAN** |
| **PC 调试** | 通常接开发板板载 USB 转串口 | **USART1**，只打日志，不参与控制 |
| **GPS** | 外接模组 | **USART6**，与运动控制独立 |

### 1.3 数据路径（谁控制谁）

```text
Jetson ──CAN2 或 RS232(V3)──► F407(仲裁) ──CAN1 0x111──► 底盘(STM32 A) ──► 电机
              ▲                              │
              │        CAN 0x211/221/…       │
              └────────── 反馈 ──────────────┘

测距模组 ──UART──► F407 滤波 ──► 避障策略 ──► 改写后再发 CAN 0x111
GPS ──USART6──► F407（CAN 模式下经 0x104~0x106 上传 Jetson，见 CAN 协议）
```

Jetson **不直接** CAN1 到底盘；测距 **不经过** Jetson，由 F407 读入后经 V3 上行带给 Jetson。

**角色分工**

| 节点 | 职责 |
|------|------|
| **Jetson** | 下发期望速度/转角/运动模式；接收安全状态、测距、底盘反馈 |
| **STM32 B** | 测距采集与滤波；心跳检测；避障仲裁；CAN 运动指令下发 |
| **STM32 A（底盘）** | 电机驱动、里程计、BMS；周期上报 CAN 反馈 |
| **四路测距模组** | E08 转接板，UART 轮询四探头 |

## 2. 硬件连接总表

| 外设 | STM32 外设 | 引脚 | 方向 | 波特率 | 代码入口 |
|------|-----------|------|------|--------|----------|
| **PC 调试串口** | USART1 | PA9=TX, PA10=RX | 双向 | **115200** 8N1 | `USART1_Init()` |
| **Jetson（默认）** | **CAN2** | **PB5=RX, PB6=TX** + 外接收发器 | 双向 | **500 kbps** 标准帧 | `JetsonCAN_Init()` |
| **Jetson（备用）** | USART2（函数名 `USART3_*`） | PA2=TX, PA3=RX | 双向 | **115200** 8N1 | `USART3_Init()`，`JETSON_LINK_CAN=0` |
| **四路测距** | USART3（中断名 `USART3_IRQHandler`） | PB10=TX/触发, PB11=RX | MCU→模组触发 + 收帧 | **9600** 8N1 | `DistanceSensor_Init()` |
| **GPS** | USART6 | PC6=TX, PC7=RX | 双向 | 上电 **9600**，配置后 **115200** | `GPS_USART6_Init()` |
| **底盘 CAN** | CAN1 | PA11=RX, PA12=TX | 双向 | **500 kbps** 标准帧 | `CAN1_Init_RangerMini()` |

### 2.1 普中麒麟 F407 原理图引脚对照（CAN / I2S / SPI）

| 原理图标签 | 引脚 | STM32 复用 | 本工程用途 | Jetson CAN 方案 |
|-----------|------|-----------|-----------|----------------|
| CAN RX/TX + 跳线帽 | PA11 / PA12 | CAN1 | **底盘 RANGER MINI** | 保持不动 |
| I2S LRCK / SCLK | PB12 / PB13 | CAN2 或 I2S | WM8978 音频（`Hardware_Check`） | **不用**（与音频冲突） |
| SPI MOSI / DCMI D5 | PB5 / PB6 | SPI1 或 **CAN2** | 启动时 PB5=SPI1 检 Flash；**仲裁模式不用摄像头** | **推荐：CAN2 RX/TX** |
| DCMI VSYNC | **PB7** | DCMI | 摄像头垂直同步（`My_DCMI_Init`） | 仲裁模式不初始化，**与 CAN2 无冲突** |

**Jetson 第二路 CAN 接线（需自购 3.3V 收发器，如 TJA1050 模块）**

```text
F407 PB6 (CAN2_TX) ──► 收发器 TXD
F407 PB5 (CAN2_RX) ◄── 收发器 RXD
收发器 CANH / CANL ──► Jetson USB-CAN（SocketCAN can1，500k）
GND 共地；两端各 120Ω 终端电阻（视总线长度，短杜邦线可暂不加）
```

> **注意**：`Hardware_Check()` 阶段 PB5 仍为 SPI1（检 W25Q128），进入 `App_MotionHwInit()` 后 `CAN2_Init_Jetson()` 会把 PB5/PB6 重配为 CAN2，与仲裁运行不冲突。

### 2.2 Jetson 链路：CAN / RS232 切换

<div style="font-size: 1.2em; line-height: 1.55; padding: 18px 22px; border-left: 6px solid #ff6f00; background: #fff8e1; margin: 12px 0;">

**⚠️ 同一固件仅启用一条 Jetson 链路** — 由编译宏选择，运行时 **不能** 同时向 CAN2 与 USART2 发 V3。

</div>

<div style="font-size: 1.15em;">

| 项目 | **CAN 模式（默认）** | **RS232 模式** |
|------|:--:|:--:|
| **宏** | `JETSON_LINK_CAN` **= 1** | **= 0** |
| **定义位置** | `APP/jetson_can/jetson_can.h` | 同上 |
| **物理接口** | **CAN2** PB5/PB6 + 3.3V 收发器 | **USART2** PA2/PA3，115200 8N1 |
| **上电初始化** | `JetsonCAN_Init()` | `USART3_Init()` |
| **协议文档** | [Jetson_CAN协议.md](./Jetson_CAN协议.md) | [Jetson_RS232协议.md](./Jetson_RS232协议.md) |
| **V3 应用层** | type 0x01/0x02/0x03，24B + XOR | **相同** |
| **扩展能力** | GPS、时间同步、故障、查询（0x104~0x10B） | **无**（仅 V3 三帧） |

</div>

#### 切换步骤（必做）

<div style="font-size: 1.25em; font-weight: 600; padding: 14px 18px; background: #f5f5f5; border-radius: 8px; margin: 10px 0;">

1. 打开 **`APP/jetson_can/jetson_can.h`**，把 **`JETSON_LINK_CAN`** 改为 **1**（CAN）或 **0**（RS232）  
2. 按上表 **重新接线**（CAN：收发器 + USB-CAN；RS232：PA2/PA3 交叉 + 共地）  
3. **全量重新编译、烧录**  
4. Jetson 侧解析器与物理接口 **必须与所选模式一致**

</div>

```c
/* APP/jetson_can/jetson_can.h */
#define JETSON_LINK_CAN  1   /* ← 改这里：1=CAN2(PB5/PB6)，0=USART2(PA2/PA3) */
```

#### RS232 接线

<div style="font-size: 1.15em; font-family: Consolas, monospace; padding: 12px 16px; background: #263238; color: #eceff1; border-radius: 6px;">

F407 PA2 (USART2_TX) ──► Jetson RX  
F407 PA3 (USART2_RX) ◄── Jetson TX  
**GND 共地**

</div>

#### 命名说明（易混淆）

- Jetson **CAN** 走 **CAN2（PB5/PB6）**；帧拼装见 `APP/jetson_can/`。
- Jetson **串口** 走 **USART2（PA2/PA3）**；编解码仍在 `APP/usart3/`（历史函数名 `USART3_*`）。
- 测距模组用的是 **USART3 硬件**（PB10/PB11），与 Jetson 串口 **无关**。

**测距触发**

- TIM4 **20 ms** 调用 `DistanceSensor_Process()`。
- 每 **12×20 ms = 240 ms** 在 PB10 输出一次低脉冲触发 E08 模组轮询四路。
- 触发前须等 `ds_rx_idx==0`，避免打断 UART 收帧。

## 3. 通信速率与周期

| 链路 | 方向 | 周期 / 速率 | 说明 |
|------|------|-------------|------|
| Jetson ↔ F407 | V3 控制/状态 | 见协议文档 | CAN：[Jetson_CAN协议.md](./Jetson_CAN协议.md)；RS232：[Jetson_RS232协议.md](./Jetson_RS232协议.md) |
| STM32 → 底盘 | CAN 0x111 运动 | **20 ms** | `ARBITER_CAN_DIV=2`，主循环 10 ms×2 |
| 底盘 → STM32 | CAN 反馈 | 底盘周期 **20 ms**（电机低速 100 ms，BMS 500 ms） | 主循环轮询 FIFO0 |
| 测距刷新 | 滤波后新数据 | **~240 ms** 触发 + 模组轮询 ~170–250 ms | 有效输出约 **0.4–0.5 Hz** |
| CAN 0x111 底盘超时 | — | **500 ms**（协议约定） | 超时底盘停车 |
| 主循环 | — | **~10 ms** | `SensorData_ShowScreen()` 末尾 `delay_ms(10)` |

## 5. CAN 协议 — 控制帧详表（STM32 B → 底盘）

### 5.0 CAN 物理层

| 项目 | 说明 |
|------|------|
| 接口 | CAN1，PA11=RX，PA12=TX |
| 帧类型 | 标准帧 11 位 ID，500 kbps |
| 字节序 | Motorola 大端（byte0=高字节） |
| 初始化 | `SJW=1, BS1=6, BS2=5, Prescaler=7` |

**发送方向**：仲裁板 → STM32 A（RANGER MINI 3.0）

---

### 5.1 ID 0x111 — 运动控制（DLC=8，周期 20 ms，超时 500 ms）

| 字节 | Bit | 信号名 | 类型 | 分辨率 | 单位 | 说明 |
|:----:|:---:|:-------|:-----|:------:|:----:|:-----|
| 0 | 7:0 | LINEAR_SPEED_H | u8 | 1 | mm/s | 线速度 MSB（有符号） |
| 1 | 7:0 | LINEAR_SPEED_L | u8 | 1 | mm/s | 线速度 LSB |
| 2 | 7:0 | SPIN_SPEED_H | u8 | 0.001 | rad/s | 自旋角速度 MSB |
| 3 | 7:0 | SPIN_SPEED_L | u8 | 0.001 | rad/s | 自旋角速度 LSB |
| 4 | 7:0 | RSV4 | u8 | — | — | 保留，发 0 |
| 5 | 7:0 | RSV5 | u8 | — | — | 保留，发 0 |
| 6 | 7:0 | STEERING_H | u8 | 0.001 | rad | 内轮转角 MSB |
| 7 | 7:0 | STEERING_L | u8 | 0.001 | rad | 内轮转角 LSB |

| 合成信号 | 公式 | 命令符号约定 |
|----------|------|--------------|
| LINEAR_SPEED | `(b0<<8)\|b1` s16 | +前进 / −后退 |
| SPIN_SPEED | `(b2<<8)\|b3` s16 | +左转 / −右转（自旋模式） |
| STEERING | `(b6<<8)\|b7` s16 | **+左转 / −右转**（阿克曼/斜移） |

| 工况 | 典型 STEERING | 典型 SPIN | 典型 V |
|------|---------------|-----------|--------|
| 阿克曼直行 | 0 | 0 | Jetson/仲裁 v |
| 阿克曼左转 | +500~+1570 | 0 | v |
| 斜移 90° 左 | +1570 | 0 | ~500 mm/s |
| 原地自旋 | 0 | ±omega | 0 |

## 6. CAN 协议 — 反馈帧详表（底盘 → STM32 B）

**接收方向**：STM32 A → 仲裁板；本工程在 `Arbiter_ProcessCANFeedback()` 中解析。

---

### 6.1 ID 0x211 — 系统状态（DLC=8，周期 20 ms）

| 字节 | Bit | 信号名 | 类型 | 分辨率 | 单位 | 说明 |
|:----:|:---:|:-------|:-----|:------:|:----:|:-----|
| 0 | 7:0 | SYSTEM_STATUS | u8 | — | — | 见表 3.1.1 |
| 1 | 7:0 | MODE_CONTROL | u8 | — | — | 见表 3.1.2 |
| 2 | 7:0 | BAT_VOLTAGE_H | u8 | 0.1 | V | 母线电压 MSB |
| 3 | 7:0 | BAT_VOLTAGE_L | u8 | 0.1 | V | 母线电压 LSB |
| 4 | 7:0 | FAULT_INFO_B3 | u8 | — | — | 故障字最高字节 |
| 5 | 7:0 | FAULT_INFO_B2 | u8 | — | — | 故障字 |
| 6 | 7:0 | FAULT_INFO_B1 | u8 | — | — | 故障字 |
| 7 | 7:0 | FAULT_INFO_B0 | u8 | — | — | 故障字最低字节 |

| 合成 | 公式 |
|------|------|
| BAT_VOLTAGE | `(b2<<8)\|b3` u16，电压 = 值 × 0.1 V |
| FAULT_INFO | `(b4<<24)\|(b5<<16)\|(b6<<8)\|b7` u32 |

#### 表 7.1.1 SYSTEM_STATUS

| 值 | 含义 |
|:--:|------|
| 0x00 | 系统正常 |
| 0x02 | 系统异常 |

#### 表 7.1.2 MODE_CONTROL

| 值 | 含义 |
|:--:|------|
| 0x00 | 待机 |
| 0x01 | CAN 指令控制 |
| 0x03 | 遥控控制 |

## 7. 四路测距模组（UART）

**连接**：USART3，PB10(TX/触发)、PB11(RX)，9600 8N1。

**帧格式**（10 字节，收满后校验）：

| 偏移 | 内容 |
|------|------|
| 0 | 0xFF 帧头 |
| 1–2 | IF1 距离 u16 BE (mm) |
| 3–4 | IF2 |
| 5–6 | IF3 |
| 7–8 | IF4 |
| 9 | 校验 = sum(byte[0..8]) & 0xFF |

**IF 与车体方向**（`main.c` 默认映射）：

| 探头 | 索引 | 方向 |
|------|------|------|
| IF1 | 0 | 前 FRONT |
| IF2 | 1 | 后 BACK |
| IF3 | 2 | 左 LEFT |
| IF4 | 3 | 右 RIGHT |

**异常值**：0xFFFF/0/0xEEEE/>59999 → 归一化为 FAILSAFE(0) 或 UNKNOWN。

详细滤波与避障逻辑见 [`测距传感器数据流.md`](测距传感器数据流.md)。

## 8. GPS（USART6）

| 项目 | 说明 |

## 10. 数据流时序（简图）

```text
TIM4 20ms ──> DistanceSensor_Process (240ms 触发测距)
                    │
USART3 IRQ ──> 10B 帧 ──> 滤波 ──> g_*_dist_mm
                    │
Jetson RX ──> CAN2 或 USART2 ──> V3 下行 ──> Arbiter_ParseJetsonCmd
                    │
main 10ms ──> Motion_ControlByDistance ──> Arbiter_Process
                    │                              │
                    ├─ CAN1 RX 轮询 <── 底盘 0x211/221/...
                    ├─ CAN1 TX 0x111 (20ms)
                    └─ Jetson TX: CAN2 0x102/103 或 USART2 0x02/0x03
```

> Jetson 物理链路由 `JETSON_LINK_CAN` 决定，见 **§2.2**。
