# 硬件连接与通信协议

本文档为 **麒麟 F407 综合实验程序** 的完整通信说明，包含 **硬件引脚连接、波特率/周期、Jetson V3 串口帧、底盘 CAN 帧** 的字节/位域级定义。

> 主要源码：`User/main.c`、`Public/usart.c`、`APP/usart3/`、`APP/distance_sensor/`、`APP/can/`、`APP/arbiter/`、`APP/gps/`

## 1. 系统架构

### 1.1 拓扑图（修正版）

原 ASCII 图有三处易误解：**① `CAN 500K` 误标在测距与 F407 之间**（CAN 只连底盘）；**② Jetson 只画了单向**（实际双向收发）；**③ PC/GPS 悬空**（实际都接在同一块 F407 上）。

                                |
    +-------------+             |             +---------------------------+

    +-------------+             |                  |      |        |

### 1.2 物理板 vs 逻辑角色

| 实体 | 是什么 | 在本图中的位置 |
|------|--------|----------------|
| **麒麟 F407 开发板** | 你手边这块实验板，程序跑在这里 | 图中中央大块 |
| **STM32 B（逻辑名）** | 代码里的「仲裁板」角色，**就是这块 F407**，不是第二块 MCU | 与 F407 同一块板 |
| **STM32 A / 底盘** | 小车上 RANGER MINI 自带的底盘控制器 | 通过 **CAN** 与 F407 相连 |
| **Jetson** | 独立上位机（如 Orin/NX） | 通过 **USART2** 与 F407 相连，**不直连底盘** |
| **四路测距** | E08 转接 + 4 探头 | 通过 **USART3** 接 F407；**不走 CAN** |
| **PC 调试** | 通常接开发板板载 USB 转串口 | **USART1**，只打日志，不参与控制 |
| **GPS** | 外接模组 | **USART6**，与运动控制独立 |

### 1.3 数据路径（谁控制谁）

```text
Jetson ──V3串口──► F407(仲裁) ──CAN 0x111──► 底盘(STM32 A) ──► 电机
                      ▲                           │
                      │     CAN 0x211/221/…       │
                      └──────── 反馈 ─────────────┘

测距模组 ──UART──► F407 滤波 ──► 避障策略 ──► 改写后再发 CAN 0x111
```

Jetson **不直接** CAN 到底盘；测距 **不经过** Jetson，由 F407 读入后通过 V3 上行帧 0x02 带给 Jetson。

**角色分工**

| 节点 | 职责 |
|------|------|
| **Jetson** | 下发期望速度/转角/运动模式；接收安全状态、测距、底盘反馈 |
| **STM32 B** | 测距采集与滤波；心跳检测；避障仲裁；CAN 运动指令下发 |
| **STM32 A（底盘）** | 电机驱动、里程计、BMS；周期上报 CAN 反馈 |
| **四路测距模组** | E08 转接板，UART 轮询四探头 |

## 2. 硬件连接总表

| 外设 | STM32 外设 | 引脚 | 方向 | 波特率 | 代码入口 |
|------|-----------|------|------|--------|----------|
| **PC 调试串口** | USART1 | PA9=TX, PA10=RX | 双向 | **115200** 8N1 | `USART1_Init()` |
| **Jetson** | USART2（函数名 `USART3_*`） | PA2=TX, PA3=RX | 双向 | **115200** 8N1 | `USART3_Init()` |
| **四路测距** | USART3（中断名 `USART3_IRQHandler`） | PB10=TX/触发, PB11=RX | MCU→模组触发 + 收帧 | **9600** 8N1 | `DistanceSensor_Init()` |
| **GPS** | USART6 | PC6=TX, PC7=RX | 双向 | 上电 **9600**，配置后 **115200** | `GPS_USART6_Init()` |
| **底盘 CAN** | CAN1 | PA11=RX, PA12=TX | 双向 | **500 kbps** 标准帧 | `CAN1_Init_RangerMini()` |

**命名说明（易混淆）**

- Jetson 相关源文件在 `APP/usart3/`，实际用的是 **USART2 硬件**（PA2/PA3）。
- 测距模组用的是 **USART3 硬件**（PB10/PB11）；触发时 PB10 临时切为 GPIO 拉低再恢复 UART。

**测距触发**

- TIM4 **20 ms** 调用 `DistanceSensor_Process()`。
- 每 **12×20 ms = 240 ms** 在 PB10 输出一次低脉冲触发 E08 模组轮询四路。
- 触发前须等 `ds_rx_idx==0`，避免打断 UART 收帧。

## 3. 通信速率与周期

| 链路 | 方向 | 周期 / 速率 | 说明 |
|------|------|-------------|------|
| Jetson → STM32 | 下行控制 | **由 Jetson 决定**（建议 ≥20 ms） | 24 字节 V3 帧；`seq` 变化才刷新心跳 |
| STM32 → Jetson | 上行状态 0x02 | **~20 ms** | 主循环 10 ms，`jetson_tx_div>=2` 且交替 |
| STM32 → Jetson | 上行扩展 0x03 | **~40 ms** | 与 0x02 交替，各类型约 40 ms 一条 |
| STM32 → 底盘 | CAN 0x111 运动 | **20 ms** | `ARBITER_CAN_DIV=2`，主循环 10 ms×2 |
| 底盘 → STM32 | CAN 反馈 | 底盘周期 **20 ms**（电机低速 100 ms，BMS 500 ms） | 主循环轮询 FIFO0 |
| 测距刷新 | 滤波后新数据 | **~240 ms** 触发 + 模组轮询 ~170–250 ms | 有效输出约 **0.4–0.5 Hz** |
| Jetson 心跳超时 | — | **300 ms** | `ARBITER_HEARTBEAT_TIMEOUT_MS` → DEGRADED |
| CAN 0x111 底盘超时 | — | **500 ms**（协议约定） | 超时底盘停车 |
| 主循环 | — | **~10 ms** | `SensorData_ShowScreen()` 末尾 `delay_ms(10)` |

## 4. 协议通用约定

| 项目 | Jetson V3 串口 | 底盘 CAN |
|------|----------------|----------|
| 物理接口 | USART2 PA2/PA3 | CAN1 PA11/PA12 |
| 速率 | 115200 8N1 | 500 kbps 标准帧 |
| 帧长度 | 固定 24 B | DLC 1~8（见各 ID） |
| 多字节整数 | **大端 BE** | **Motorola 大端**（byte0=高字节） |
| 有符号 16 位 | s16 BE | s16 BE |
| 有符号 32 位 | — | s32 BE（byte0 最高字节） |
| 校验 | byte[23]=XOR(byte[0..22]) | CAN 硬件 CRC |

**XOR 校验示例（Jetson）**

```text
chk = 0
for i in 0..22: chk ^= frame[i]
frame[23] = chk
```

### 5.2 下行控制帧 type=0x01（24 字节）

|:----:|:---:|:-------|:-----|:------------|:-----|

| 1 | 7:0 | FRAME_TYPE | u8 | 固定 **0x01** | 下行控制 |
| 2 | 7:0 | SEQ | u8 | 0~255 循环 | **变化时刷新心跳**（300 ms 超时） |

| 4 | 7:0 | V_H | u8 | — | 线速度高字节 |

| 6 | 7:0 | OMEGA_H | u8 | — | 角速度高字节 |
| 7 | 7:0 | OMEGA_L | u8 | — | 角速度低字节 |
| 8 | 7:0 | STEER_H | u8 | — | 转角高字节 |
| 9 | 7:0 | STEER_L | u8 | — | 转角低字节 |
| 10 | 7:0 | MOTION_MODEL | u8 | 见表 1.2.2 | 运动模型 → CAN 0x141 |
| 11 | 7:0 | LIGHT_EN | u8 | 0/1 | 灯光使能 → CAN 0x121 byte0 |
| 12 | 7:0 | LIGHT_MODE | u8 | 0/1 | 灯光模式 → CAN 0x121 byte1 |
| 13 | 7:0 | CLEAR_ERROR | u8 | 见表 B.4 | 非 0 触发 CAN 0x441 |
| 14 | 7:0 | RSV14 | u8 | 0 | 保留 |
| 15 | 7:0 | RSV15 | u8 | 0 | 保留 |
| 16 | 7:0 | RSV16 | u8 | 0 | 保留 |
| 17 | 7:0 | RSV17 | u8 | 0 | 保留 |
| 18 | 7:0 | RSV18 | u8 | 0 | 保留 |
| 19 | 7:0 | RSV19 | u8 | 0 | 保留 |

| 22 | 7:0 | RSV22 | u8 | 0 | 保留 |

**组合字段解析**

| 信号名 | 合成公式 | 类型 | 单位 | 正方向 |
|--------|----------|------|------|--------|
| V | `(byte4<<8)\|byte5` | s16 | mm/s | +前进 / −后退 |
| OMEGA | `(byte6<<8)\|byte7` | s16 | 0.001 rad/s | +左转 / −右转 |
| STEER | `(byte8<<8)\|byte9` | s16 | 0.001 rad | +左转 / −右转 |

#### 表 5.2.1 MODE_REQ

| 值 | 含义 | STM32 动作 |
|:--:|------|------------|
| 0x00 | 待机 | 发 CAN 0x421 = 0x00 |
| 0x01 | CAN 指令控制 | 发 CAN 0x421 = 0x01 |
| 0x02 | 遥控（请求） | 发 CAN 0x421 = 0x00（待机） |

#### 表 5.2.2 MOTION_MODEL

| 值 | 含义 | CAN 0x141 |
|:--:|------|-----------|
| 0x00 | 前后阿克曼 | 0x00 |
| 0x01 | 斜移 | 0x01 |
| 0x02 | 自旋 | 0x02 |
| 0x03 | 驻车 | 0x03 |

> **注意**：byte4~9 的 v/omega/steer 经 **避障仲裁** 后才写入 CAN 0x111，NORMAL 模式可近似透传，SPEED_LIMIT/DEGRADED 会改写。

**组合字段**

| 信号名 | 合成 | 类型 | 单位 | 备注 |
|--------|------|------|------|------|
| FB_V | `(b6<<8)\|b7` | s16 | mm/s | 来自 0x221 |
| FB_OMEGA | `(b8<<8)\|b9` | s16 | 0.001 rad/s | 来自 0x221 |
| FB_STEER | `(b10<<8)\|b11` | s16 | 0.001 rad | 来自 0x221；**符号与 0x111 命令相反** |
| SONAR_* | `(bH<<8)\|bL` | u16 | mm | 0xFFFF=无效/未更新 |
| BAT_V | `(b20<<8)\|b21` | u16 | 0.1 V | 实际电压 = 值 / 10 |

#### 表 5.3.1 SAFETY_STATE

| 值 | 仲裁模式 | 含义 |
|:--:|----------|------|
| 0x01 | NORMAL | 正常，可透传 Jetson |
| 0x02 | SPEED_LIMIT | 限速/避障 |
| 0x03 | DEGRADED / RECOVERING | Jetson 心跳丢失或恢复中 |
| 0x04 | EMERGENCY | 紧急停车 |

#### 表 5.3.2 LINK_STATE 位定义

| Bit | 名称 | =1 时含义 |
|:---:|------|-----------|
| 0 | JETSON_HB_LOST | Jetson 心跳超时（>300 ms 无新 seq） |
| 1 | CHASSIS_FAULT | 底盘 0x211 byte0 == 0x02 系统异常 |
| 2~7 | — | 保留，读 0 |

#### 表 5.3.3 LIMIT_FACTOR 算法（SPEED_LIMIT 时）

```text
若 nearest_dist <= 60 mm  → 0
若 nearest_dist >= 180 mm → 100
否则 线性: (dist - 60) * 100 / (180 - 60)
```

## 6. CAN 协议 — 控制帧详表（STM32 B → 底盘）

**发送方向**：仲裁板 → STM32 A（RANGER MINI 3.0）

---

### 6.1 ID 0x111 — 运动控制（DLC=8，周期 20 ms，超时 500 ms）

| 字节 | Bit | 信号名 | 类型 | 分辨率 | 单位 | 说明 |
|:----:|:---:|:-------|:-----|:------:|:----:|:-----|
| 0 | 7:0 | LINEAR_SPEED_H | u8 | 1 | mm/s | 线速度 MSB（有符号） |
| 1 | 7:0 | LINEAR_SPEED_L | u8 | 1 | mm/s | 线速度 LSB |
| 2 | 7:0 | SPIN_SPEED_H | u8 | 0.001 | rad/s | 自旋角速度 MSB |
| 3 | 7:0 | SPIN_SPEED_L | u8 | 0.001 | rad/s | 自旋角速度 LSB |
| 4 | 7:0 | RSV4 | u8 | — | — | 保留，发 0 |
| 5 | 7:0 | RSV5 | u8 | — | — | 保留，发 0 |
| 6 | 7:0 | STEERING_H | u8 | 0.001 | rad | 内轮转角 MSB |
| 7 | 7:0 | STEERING_L | u8 | 0.001 | rad | 内轮转角 LSB |

| 合成信号 | 公式 | 命令符号约定 |
|----------|------|--------------|
| LINEAR_SPEED | `(b0<<8)\|b1` s16 | +前进 / −后退 |
| SPIN_SPEED | `(b2<<8)\|b3` s16 | +左转 / −右转（自旋模式） |

## 8. 四路测距模组（UART）

**连接**：USART3，PB10(TX/触发)、PB11(RX)，9600 8N1。

**帧格式**（10 字节，收满后校验）：

| 偏移 | 内容 |
|------|------|
| 0 | 0xFF 帧头 |
| 1–2 | IF1 距离 u16 BE (mm) |
| 3–4 | IF2 |
| 5–6 | IF3 |
| 7–8 | IF4 |
| 9 | 校验 = sum(byte[0..8]) & 0xFF |

**IF 与车体方向**（`main.c` 默认映射）：

| 探头 | 索引 | 方向 |
|------|------|------|
| IF1 | 0 | 前 FRONT |
| IF2 | 1 | 后 BACK |
| IF3 | 2 | 左 LEFT |
| IF4 | 3 | 右 RIGHT |

**异常值**：0xFFFF/0/0xEEEE/>59999 → 归一化为 FAILSAFE(0) 或 UNKNOWN。

详细滤波与避障逻辑见 [`测距传感器数据流.md`](测距传感器数据流.md)。

## 14. Jetson 侧对接建议

1. **下行**：至少 **20 ms** 发一帧 0x01，**seq 递增**；超过 **300 ms** 无新 seq → STM32 进 DEGRADED 本地避障。
2. **上行**：每 20 ms 会收到 0x02 或 0x03 之一；需同时解析两种 type 才能获得完整状态。
3. **单位**：速度 mm/s，角度/角速度 ×1000（0.001 rad / rad/s）。
4. **模式**：`mode_req=1` 且底盘 0x211 为 CAN 控制时，0x111 才会真正驱动车辆；遥控接管时 STM32 主动发零速。
5. **测距**：四路 u16，0xFFFF 表示无效/未更新。
